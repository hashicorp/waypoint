# The `waypoint logs` Command

From the same application directory, you can execute the `waypoint logs` command to inspect the running logs for your deployment.

```bash
waypoint logs
```

You should see an output similar to the below.

```shell
2020-09-16T23:52:15.477Z uaubxwa: /docker-entrypoint.sh: /docker-entrypoint.d/ is not empty, will attempt to perform configuration
2020-09-16T23:52:15.477Z uaubxwa: /docker-entrypoint.sh: Looking for shell scripts in /docker-entrypoint.d/
2020-09-16T23:52:15.479Z uaubxwa: /docker-entrypoint.sh: Launching /docker-entrypoint.d/10-listen-on-ipv6-by-default.sh
2020-09-16T23:52:15.494Z uaubxwa: 10-listen-on-ipv6-by-default.sh: Getting the checksum of /etc/nginx/conf.d/default.conf
2020-09-16T23:52:15.512Z uaubxwa: 10-listen-on-ipv6-by-default.sh: error: /etc/nginx/conf.d/default.conf differs from the packages version

2020-09-16T23:52:15.519Z uaubxwa: /docker-entrypoint.sh: Configuration complete; ready for start up
```
