# Waypoint Init

Command: `waypoint init`

Initialize and validate a project

@include "commands/init_desc.mdx"

## Usage

Usage: `waypoint init [options]`

#### Global Options

- `-plain` - Plain output: no colors, no animation.
- `-app=<string>` - App to target. Certain commands require a single app target for Waypoint configurations with multiple apps. If you have a single app, then this can be ignored.
- `-workspace=<string>` - Workspace to operate in.

#### Command Options

- `-from-project=<string>` - Create a new application by fetching the given application froma remote source or from a local project folder or file on disk.
- `-into=<string>` - Where to write the application fetched via -from-project
- `-update` - Update the project configuration if it already exists. This can be used to update settings such as the remote runner data source.
