## Markup

1. Explicitly define the `type` attribute on the `<input>` element.
    - This will provide the information necessary to write CSS that explicitly
      targets specific input types.
2. Define a unique `id` attribute on the `<input>` element.
    - This will be used to explicitly associate the input with a semantic
      `<label>` via the `[for]` attribute.

```html
<input
  type="text"
  id="{ID}"
  class="pds-formControl"
/>
```


## Styling
Styles are scoped to the `input.pds-formControl` selector.

```scss
@use "pds/components/text-input";
```

## Similar Inputs
The following input types are all text-like in that if a browser doesn't
support the given type, it will render the element as if it were
`<input type="text">`.

- Date
- Datetime
- Email
- Month
- Number
- Password
- Search
- Telephone
- Time
- URL
- Week
