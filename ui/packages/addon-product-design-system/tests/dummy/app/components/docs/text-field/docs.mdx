# Text Field

Single-line text entry with possible error message and optional help text.

- [Markup](#markup)
- [Styling](#styling)
- [Accessibility](#accessibility)
- [See Also](#see-also)


## Markup

1. Apply the `.pds-formField` class to a `<div>` around text field children.

Children should appear in the following order:

1. [Field Name](?path=/docs/components-field-name--index)
2. [Help Text](?path=/docs/components-help-text--index)
3. [Text Input](?path=/docs/components-text-input--index)
4. [Error Message](?path=/docs/components-error-message--index)

For the majority of use cases, this can be achieved by defining markup in
explicit DOM order.  Children of `.pds-formField` are arranged vertically,
in DOM order.

```html
<div class="pds-formField">
  <label for="{INPUT_ID}" class="pds-fieldName">...</label>

  <p id="{HELP_ID}" class="pds-helpText">...</p>

  <input
    class="pds-formControl"
    id="{INPUT_ID}"
    type="text"
    aria-describedby="{HELP_ID} {ERROR_ID}"
  />

  <Pds::ErrorMessage id="{ERROR_ID}">...</Pds::ErrorMessage>
</div>
```


## Styling
Styles are scoped to the `.pds-formField` selector.

```scss
@use "pds/components/text-field";
```
