# Radio

## Radio Control

__NOTE:__ `.radioControl` class used to correlate markup with later documentation.

```html
<div class="radioControl">
  <input
    type="radio"
    id="control"
    name="..."
    aria-describedby="help"
  />

  <label for="control">
    Control Label
  </label>

  <p id="help">
    Help text goes here.
  </p>
</div>
```

### Radio Control Elements

Checkbox Control elements should appear in the following order:

1. Control
2. Label
3. Help Text

For the majority of use cases, this can be achieved by defining
markup in explicit DOM order.



## Radio Group

__NOTE:__ `.radioControl` class used only to correlate nested markup with
previous section documentation.


```html
<fieldset>
  <!-- 1. -->
  <legend>Field Name</legend>

  <!-- 2. -->
  <div>
    <div class="radioControl">
      <input
        type="radio"
        id="option1"
        name="..."
        aria-describedby="help1 errors"
      />

      <label for="option1">
        Option 1 Label
      </label>

      <p id="help1">
        Help text goes here.
      </p>
    </div>
    ...
    <div class="radioControl">
      <input
        type="radio"
        id="optionN"
        name="..."
        aria-describedby="helpN errors"
      />

      <label for="optionN">
        Option N Label
      </label>

      <p id="helpN">
        Help text goes here.
      </p>
    </div>
  </div>

  <!-- 3. -->
  <ul id="errors">
    <li>Error message 1</li>
    ...
    <li>Error message N</li>
  </ul>
</fieldset>
```

### Radio Group Elements

A radio group should display elements in the following order (left-to-right, top-to-bottom):

1. Field Name / Legend
    - Use `<legend>` to provide fielset information to assistive technologies.
2. [Radio Controls](#radio-control) (wrapped in a DIV)
    - See ["Forms: Fieldsets"](?path=/docs/guides-forms-overview--page#fieldsets) for more information about the extra `<div>`
3. Field Errors
    - Associate each radio control with the _entire_ error list_(`<ul>`) using `[aria-describedby]` on the `<input>`.
    - All errors should apply to the entire radio group.

For the majority of use cases, this can be achieved by defining
markup in explicit DOM order.
